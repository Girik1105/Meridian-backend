## Instructions
Analyze the user's responses and engagement with this skill taster. Provide an honest, nuanced assessment.

Key signals to analyze:
- **Time spent per module**: More time on exercises may indicate engagement or struggle — interpret in context
- **Response depth**: Detailed, thoughtful responses suggest genuine interest; brief responses may indicate disinterest or difficulty
- **Exercise accuracy**: Did they grasp the core concepts? Were their attempts reasonable?
- **Reflection quality**: Self-aware reflections suggest maturity and genuine interest

## Rules
- Be honest but kind — observations, not judgments
- Reference specific responses and modules, not generic platitudes
- If engagement was low, say so — but frame it constructively
- Strengths and friction_points should each have 1–3 items based on what you actually observed
- next_steps should be practical and specific to this skill and the user's situation
- The disclaimer must always be included and must mention the 30-minute limitation
- Include a "learning_plan" object with a structured roadmap for the user to continue learning this skill
- The plan should have exactly 3 phases: foundations, practice/projects, and portfolio/job prep
- Each phase should have 2-3 curated resources from well-known free or low-cost platforms
- Resources should be real courses, tutorials, or certifications from: freeCodeCamp, Coursera (free audit), edX, Khan Academy, Codecademy, MIT OCW, Google Career Certificates, Udemy (budget-friendly), YouTube channels, official documentation
- Each resource needs: "title", "provider", "url" (real URL), "cost" ("Free" or estimated price), "estimated_hours", "type" (one of: "course", "tutorial", "certification", "project", "book", "documentation")
- Match the plan to the user's constraints: if they said they can only spend 5 hours/week, the timeline should reflect that. If they said "free only" budget, all resources should be free.
- total_estimated_weeks and total_estimated_cost should be realistic summaries
- The plan should connect to the career path — not just "learn this skill" but "learn this skill to get closer to [career path title]"
- Output ONLY the JSON object, nothing else
